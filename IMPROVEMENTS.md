# Что можно улучшить в `etch`

Результат аудита кодовой базы (1310 строк в `src/main.rs`, без тестов, с несколькими багами и узкими местами).

---

## 🔴 Критические баги и логика

- [ ] **Не воспроизводимость на CPU**  
  `device.set_seed(seed)` вызывается только для GPU. На CPU `Tensor::randn` использует свой внутренний PRNG, и `--seed` не работает.

- [ ] **Захардкожен `guidance_scale = 4.0` для FLUX**  
  Для `schnell` guidance не нужен вообще — лучше передавать `None`. Для `dev` стоит сделать CLI-флаг.

- [ ] **Отсутствие валидации `--height` / `--width`**  
  VAE требует кратности 8 (или 16). Невалидные значения ломают инференс на этапе `unpack`/`decode` с непонятной ошибкой.

## ⚡ Производительность

- [x] **Параллельная загрузка T5 и CLIP**  
  Они независимы, но грузятся последовательно. `std::thread::scope` или `rayon` ускорят старт.

- [x] **Переход с `F32` на `F16` / `BF16`**  
  PR #9: `--dtype bf16|f16|f32`. По умолчанию `bf16` на Metal, `f32` на CPU. Сокращает потребление памяти вдвое.

- [x] **Ленивая загрузка VAE в SDXL + `--vae-cpu`**  
  PR #9: VAE грузится только после UNet-цикла. `--vae-cpu` форсирует декодирование на CPU — полезно на машинах с tight memory.

- [ ] **GGUF-loader: убрать лишний поток**  
  `load_gguf_with_spinner` запускает отдельный `thread` только ради спиннера. Достаточно `print!("Loading...")` → `load()` → `println!("done")`.

## 🎨 UX и CLI

- [ ] **Заменить кастомный прогресс-бар на `indicatif`**  
  Сейчас прогресс дублируется в `run_flux` и `run_sdxl` через ручной `print!`. `indicatif` даст единый стиль, ETA и spinner внешне.

- [ ] **Добавить фидбек при скачивании с HuggingFace**  
  Первый запуск качает 10–24 GB без сообщений. Стоит выводить `Downloading {file}...` перед `repo.get()`.

- [ ] **Добавить `--verbose` флаг**  
  Включать `target` и `time` в `tracing_subscriber` для отладки.

- [ ] **Валидация `--lora-scale`**  
  Документация говорит "0.0–1.0", но код принимает любое `f64`.

- [ ] **Валидация `--guidance-scale`**  
  Принимает отрицательные значения без ошибки; результат — некорректный CFG.

- [ ] **Валидация `--clip-skip`**  
  Значения больше количества слоёв приведут к панике или некорректной индексации.

- [x] **Предупреждение при GGUF + Metal**  
  PR #9: `info!("GGUF mode: DiT runs on CPU")` при `is_gguf`.

- [ ] **Явный флаг `--seed` в логе**  
  `Seed: 1234` логируется одинаково для `--seed 1234` и авто-сгенерированного. Лучше: `Seed: 1234 (auto)` или `Seed: 1234 (explicit)`, чтобы было понятно, можно ли воспроизвести.

- [ ] **Валидация формата LoRA-файла при загрузке**  
  Если передать не LoRA-файл (случайный `.safetensors`), ошибка появляется только глубоко в `apply_lora` как "matched 0 layers". Лучше проверить наличие ключей `*.lora_down.weight` сразу после загрузки и выдать понятное сообщение.

## 🛡 Надёжность

- [ ] **Логирование провальных запусков в `log.jsonl`**  
  Сейчас запись в лог происходит только при успехе. Если инференс упал — промпт и сид теряются. Добавить `"status": "failed", "error": "..."` при ошибке.

- [ ] **Логирование приблизительного размера загруженных моделей**  
  После загрузки T5, CLIP, DiT, UNet, VAE — выводить `info!("T5: ~9.5 GB loaded")` через размер файла. Помогает пользователям понять, почему OOM и откуда.

- [ ] **Проверка каналов в `save_image`**  
  ```rust
  let (_c, h, w) = img.dims3()?;
  ```  
  `_c` игнорируется. Если модель вернёт не RGB, `image::save_buffer(..., Rgb8)` запишет мусор. Нужен `ensure!(c == 3)`.

- [ ] **Безопасный `append` в `log.jsonl`**  
  При параллельных запусках несколько процессов могут испортить файл. Добавить `fd-lock` или `fs2::FileExt::lock_exclusive`.

- [ ] **Graceful shutdown по `Ctrl+C`**  
  Добавить обработчик `ctrlc` для корректного прерывания и досрочного освобождения памяти (особенно важно на 32 GB машинах).
